Answer: A microcontroller has CPU, memory, and I/O on a single chip. It is used in control-based applications like washing machines. It is cheaper and handles repetitive tasks. A microprocessor is only CPU and needs external memory. It is mainly used in general-purpose computers.

Answer: 8051 has CPU, ROM, RAM, timers, serial port, and I/O ports. ROM stores program, RAM stores temporary data. Timers are used for delays and counting. I/O ports allow external communication. All parts are linked by buses.

Answer: The PC holds the address of the next instruction. It ensures program flow is correct. The DPTR is a 16-bit register for external memory. It stores data/code memory addresses. Both are essential for execution.

Answer: Addressing modes tell CPU how to access data. 8051 has Immediate, Register, Direct, Indirect, and Indexed modes. Immediate uses constant values in instruction. Direct/Indirect use memory or registers. Indexed is used for program memory.

Answer: 8051 has 128B RAM and 4KB ROM inside. RAM is split into register banks, bit memory, and general data. ROM stores program instructions. External memory up to 64KB can be added. This makes 8051 flexible for projects.

Answer: SFRs control microcontroller features. They manage I/O ports, timers, and CPU status. Each has a fixed address in memory. Examples: ACC, B, PSW, SP. They let programmers control hardware directly.

Answer: Interrupts pause execution to handle urgent tasks. 8051 has five interrupts: 2 external, 2 timer, 1 serial. Each has a service routine (ISR). They are enabled using IE register. This allows real-time response.

Answer: Accumulator is used for arithmetic, logic, and data transfer. It is the main register in 8051. B register works with Accumulator for multiplication/division. It can also be used as general-purpose. Together, they speed up operations.

Answer: Watchdog Timer prevents system hangs. It resets the MCU if program freezes. Software must refresh it regularly. If not refreshed, MCU restarts. It ensures system reliability.

Answer: Embedded systems use ROM, RAM, and Flash. ROM stores permanent programs. RAM stores temporary data. Flash allows reprogramming of firmware. Each memory type has a role.
