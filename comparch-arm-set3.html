Placement Q&A - Set 3 (Communication Protocols)

Answer: UART means Universal Asynchronous Receiver Transmitter. It sends and receives data serially without a clock. Data is framed with start and stop bits. It is simple and widely used for short-distance communication. Example: Serial communication in Arduino.

Answer: SPI is Serial Peripheral Interface. It uses four lines: MISO, MOSI, SCK, and SS. It is faster than UART and I¬≤C. One master can connect to many slaves. It is used for sensors, displays, and memory chips.

Answer: I¬≤C is Inter-Integrated Circuit communication. It uses only two wires: SDA and SCL. It allows multiple masters and slaves. It is slower than SPI but saves pins. It is used in EEPROMs, sensors, and RTC modules.

Answer: UART uses two wires and is asynchronous. SPI uses four wires and is faster. I¬≤C uses two wires and supports multiple devices. UART is simple, SPI is fast, and I¬≤C saves pins. Each is used in different applications.

Answer: CAN stands for Controller Area Network. It is a robust serial protocol for automotive and industrial systems. It supports multiple nodes without a master. It has error detection and priority messaging. Example: Used in cars for ECU communication.

Answer: USB means Universal Serial Bus. It connects peripherals like keyboards, storage, and printers. It supports plug-and-play and hot swapping. Data transfer is much faster than UART. It is widely used in computers and embedded devices.

Answer: Synchronous communication uses a common clock signal. Data is sent in sync with the clock. Asynchronous does not use a clock but uses start/stop bits. Synchronous is faster but complex. Asynchronous is simpler and cheaper.

Answer: Baud rate is the speed of data transfer. It defines symbols per second in UART. Example: 9600 baud means 9600 bits/sec. Both sender and receiver must match baud rate. Otherwise, data errors occur.

Answer: Parity is an error-checking method. It adds a bit to make data even or odd. Receiver checks parity to detect errors. It is simple but not very strong. Used in UART and old serial protocols.

Answer: IoT devices use MQTT, CoAP, HTTP, and Bluetooth Low Energy. MQTT is lightweight for sensors. CoAP is designed for constrained devices. HTTP is common but heavy. BLE saves energy in wearables. Choice depends on the application.
