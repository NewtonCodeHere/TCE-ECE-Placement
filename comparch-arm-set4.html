Placement Q&A - Set 4 (RTOS & Embedded Concepts)

Answer: RTOS means Real-Time Operating System. It manages tasks with strict timing. It ensures deadlines are met without delay. It provides multitasking, semaphores, and scheduling. Used in robotics, medical, and automotive systems.

Answer: A semaphore controls access to shared resources. It prevents conflicts between tasks. Only one task can use a resource at a time. Other tasks wait until it is free. It is key in multitasking systems.

Answer: Mutex means Mutual Exclusion. It is like a semaphore but ownership is strict. Only the task that locked it can unlock it. This avoids deadlocks and data corruption. Used widely in real-time applications.

Answer: Tasks are independent units in RTOS with their own stack. Threads are lighter, sharing memory inside one process. Tasks are heavier than threads. Both allow multitasking. RTOS schedules them based on priority.

Answer: A critical section is code that accesses shared data. If two tasks enter, errors may happen. It is protected using semaphores or mutexes. This ensures only one task enters at a time. It keeps data safe.

Answer: Watchdog Timer prevents system hangs. The program resets it regularly. If not, it assumes the program is stuck. Then it resets the system automatically. This improves reliability of devices.

Answer: A bootloader runs at power-on. It initializes hardware like memory and clock. It loads the main program into CPU. It also supports firmware updates. Bootloaders make devices upgradable.

Answer: Polling checks devices regularly for data. It wastes CPU time if no data. Interrupts notify CPU only when events occur. Interrupts are more efficient than polling. They improve system response time.

Answer: Embedded devices often run on batteries. Low power extends operating life. Sleep modes save energy when idle. Efficient software reduces CPU usage. Important for IoT and portable devices.

Answer: FOTA means firmware update wirelessly. Device downloads new firmware via network. Bootloader verifies and installs it. Old code is replaced with new one. It allows remote maintenance of devices.
