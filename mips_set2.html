Answer: MIPS pipelining can encounter three types of hazards:

  • Data hazards: Occur when instructions depend on the results of previous instructions.
  • Control hazards: Arise from branch and jump instructions affecting instruction flow.
  • Structural hazards: Happen when hardware resources are insufficient for concurrent execution.

These are managed using techniques like forwarding, stalling, and branch prediction.

Answer: MIPS follows a load/store architecture where:

  • Arithmetic and logical operations are performed only on registers.
  • Data must be explicitly loaded from memory into registers using load instructions.
  • Results are stored back to memory using store instructions.

This separation simplifies instruction execution and enhances pipeline efficiency.

Answer: The key differences are:

  • Branch instructions: Perform conditional updates to the Program Counter (PC) based on register values (e.g., beq, bne).
  • Jump instructions: Perform unconditional updates to the PC to a specified address (e.g., j, jal).

Answer: The delay slot is the instruction immediately following a branch or jump. It is executed before the branch takes effect, allowing:

  • Better pipeline utilization.
  • Reduced control hazards.

Compilers often place useful instructions in the delay slot to optimize performance.
