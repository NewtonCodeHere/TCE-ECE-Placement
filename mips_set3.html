Answer: MIPS primarily uses base+offset addressing for memory operations. The effective memory address is calculated by:

  • Adding a constant offset to the contents of a base register.
  • Used in instructions like lw (load word) and sw (store word).

This approach simplifies hardware design and supports efficient memory access.

Answer: MIPS handles interrupts and exceptions using a structured mechanism:

  • When an exception occurs, control is transferred to a predefined handler.
  • The EPC (Exception Program Counter) register stores the address of the interrupted instruction.
  • Special registers and coprocessor 0 are used to manage exception types and responses.

This ensures reliable recovery and continuation of program execution.

Answer: MIPS architecture supports coprocessors to extend functionality:

  • Coprocessor 0: Handles system control tasks like exception handling and memory management.
  • Coprocessor 1: Performs floating-point arithmetic operations.

These units operate alongside the main CPU to offload specialized tasks and improve performance.

Answer: Branching in MIPS alters the flow of execution based on conditions:

  • beq $t0, $t1, label: Branches to label if $t0 == $t1.
  • bne $t0, $t1, label: Branches to label if $t0 != $t1.

These instructions help implement loops, conditionals, and decision-making logic in programs.

Answer: The HI and LO registers store results from multiplication and division operations:

  • mult and div instructions place results in HI and LO.
  • mfhi and mflo are used to move values from these registers to general-purpose registers.

This separation allows handling of 64-bit results in a 32-bit architecture.
