Answer: Delay slot hazards occur because the instruction in the delay slot executes regardless of the branch outcome. This can lead to unintended behavior if the delay slot contains an instruction that shouldn't run.

  • MIPS handles this by requiring compilers to carefully select instructions for the delay slot.
  • Often, a useful instruction is placed there to optimize performance.
  • If no safe instruction is available, a nop (no operation) is inserted.

Answer: When an exception occurs in MIPS:

  • The current Program Counter (PC) is saved to the EPC register.
  • Interrupts are disabled to prevent further disruption.
  • Control is transferred to a fixed exception handler address.

This allows the system to diagnose and respond to the exception before resuming normal execution.

Answer: The stack in MIPS is managed using the $sp (stack pointer) register:

  • The stack grows downward in memory (toward lower addresses).
  • push operations are implemented using sw (store word).
  • pop operations use lw (load word).

This structure supports function calls, local variables, and return address storage.

Answer: MIPS-32 includes a dedicated floating-point coprocessor (Coprocessor 1) that supports:

  • Single-precision (32-bit) operations.
  • Double-precision (64-bit) operations.
  • Instructions like add.s, mul.d, div.s, etc.

This enables efficient handling of scientific and multimedia computations.

Answer: microMIPS is a compressed instruction set variant of MIPS-32 designed to reduce code size:

  • Uses a mix of 16-bit and 32-bit instructions.
  • Maintains compatibility with standard MIPS-32 architecture.
  • Ideal for embedded systems where memory footprint is critical.

It offers similar performance while improving code density and reducing memory usage.
