Switch Level Modelling- Set 2

Answer: Switch-level modelling represents circuits at the transistor (MOSFET) level using primitives like nmos, pmos, cmos, and TG. It is used for simulating custom digital designs at the transistor level, such as standard cells or pass-transistor logic.

Answer:

  • Behavioral: High-level constructs (if, case), abstracts hardware.
  • Gate-level: Uses logic gates (and, or, not).
  • Switch-level: Lowest abstraction using transistors (nmos, pmos).
  • nmos: NMOS transistor (control, two nodes).
  • pmos: PMOS transistor (control, two nodes).
  • cmos: Complementary switch (NMOS + PMOS for CMOS logic).
  • tran: Unconditional bidirectional pass switch.
  • tranif1: Conducts when control = 1.
  • tranif0: Conducts when control = 0.

Answer:

The output can be: • 0, 1: logic levels • z: high-impedance (floating) • x: unknown, due to contention or uninitialized inputs • Possibly weak1, strong0, etc., internally (depending on drivers)

module transmission_gate (out, in, EN, EN_bar);
  input in, EN, EN_bar;
  output out;
  wire out;
  nmos n1 (out, in, EN);
  pmos p1 (out, in, EN_bar);
endmodule
            

module nmos_switch (Y, A, C);
  input A, C;
  output Y;
  wire Y;
  nmos n1 (Y, A, C);
endmodule
            
 
module sr_latch (Q, Qbar, S, R);
  input S, R;
  output Q, Qbar;
  wire Q, Qbar;
  nmos n1 (Qbar, 1'b0, S); 
  nmos n2 (Q, 1'b0, R);
  pullup (Q);
  pullup (Qbar);
endmodule
